1 · Executive Summary

A SQL Injection vulnerability (CVE-2024-1234) was identified in the authentication endpoint /api/v1/auth/login of the fintech application startup-exemplo.com.br. The vulnerability allows an unauthenticated attacker to bypass authentication, extract arbitrary data from the backend database, and potentially escalate to remote code execution.

The vulnerability received a CVSS 3.1 Base Score of 9.8 (Critical) with Critical severity. Approximately 12,450 user records were potentially exposed, including usernames, password hashes (BCrypt), email addresses, and partial CPF numbers.

Under LGPD (Lei 13.709/2018), Arts. 44 and 48, the data controller must notify affected data subjects and the ANPD within a reasonable timeframe.

4 · Detailed Finding — CVE-2024-1234

Description

The /api/v1/auth/login endpoint constructs SQL queries using string concatenation of user-supplied input, enabling an attacker to inject arbitrary SQL.
