## Brand Overview

**Product Name:** TaskFlow

**Tagline:** "Where Work Flows Naturally"

**Positioning Statement:** TaskFlow is the project management platform that helps mid-market tech teams streamline workflows, reduce context switching, and deliver projects on time — without the learning curve.

**Brand Personality:**
- **Professional** — credible, data-driven, trustworthy
- **Approachable** — warm, conversational, never corporate-stiff
- **Modern** — forward-thinking, innovative, design-led
- **Action-oriented** — focuses on outcomes, not features

**Brand Values:**
1. Simplicity over complexity
2. Transparency in communication
3. Empowerment through clarity
4. Continuous improvement

## Logo Usage

### Primary Logo
- Format: SVG (preferred) or PNG with transparent background
- Minimum width: 120px for digital, 25mm for print
- Clear space: equal to the height of the "T" in TaskFlow on all sides

### Logo Variations

| Version | Background | File |
|---------|-----------|------|
| Full color | Light backgrounds | `taskflow-logo-full.svg` |
| Reversed (white) | Dark backgrounds | `taskflow-logo-white.svg` |
| Monochrome | Single-color print | `taskflow-logo-mono.svg` |
| Icon only | Favicons, avatars | `taskflow-icon.svg` |

### Incorrect Usage
- Do NOT stretch or distort the logo
- Do NOT add shadows, outlines, or gradients to the logo
- Do NOT place on busy backgrounds without the reversed version
- Do NOT rotate or skew the logo
- Do NOT change the brand colors in the logo

## Typography

### Primary Typeface: Inter (Google Fonts)
- **Headings:** Inter Bold / SemiBold
- **Body Text:** Inter Regular / Medium
- **Labels/UI:** Inter SemiBold, uppercase with letter-spacing

```css
--font-primary: 'Inter', -apple-system, BlinkMacSystemFont, sans-serif;

h1 {
    font-size: 3rem;
    font-weight: 700;
    line-height: 1.1;
    letter-spacing: -0.02em;
}

h2 {
    font-size: 2rem;
    font-weight: 600;
    line-height: 1.2;
    letter-spacing: -0.01em;
}

body {
    font-size: 1rem;
    font-weight: 400;
    line-height: 1.6;
}

small, label {
    font-size: 0.75rem;
    font-weight: 600;
    text-transform: uppercase;
    letter-spacing: 0.05em;
}
```

### Font Scale
| Level | Size | Weight | Use |
|-------|------|--------|-----|
| H1 | 48px (3rem) | 700 | Page titles, hero headlines |
| H2 | 32px (2rem) | 600 | Section titles |
| H3 | 24px (1.5rem) | 600 | Card titles |
| Body | 16px (1rem) | 400 | Paragraph text |
| Small | 14px (0.875rem) | 400 | Captions, footnotes |
| Micro | 12px (0.75rem) | 600 | Labels, badges |

### Hierarchy Rules
- Maximum 3 levels of heading visible per section
- H1 should only appear once per page
- Use font size and weight to create visual hierarchy, not color alone

## Voice & Tone

### Voice (Always On)
Our voice is **clear, confident, and helpful**. We speak like a knowledgeable colleague — not a textbook and not a marketing robot.

### Tone (Context-Dependent)

| Context | Tone | Example |
|---------|------|---------|
| Landing page | Confident, benefit-forward | "Ship faster. Stress less. TaskFlow brings your team's work together in one place." |
| Blog post | Educational, conversational | "We tested 12 project management tools so you don't have to. Here's what actually works." |
| Social media | Energetic, relatable | "That feeling when all your tasks are done by 3pm. That's TaskFlow energy. 😎" |
| Error message | Helpful, reassuring | "Oops, that didn't go through. No data was lost — try again?" |
| Email nurture | Personal, valuable | "Hi Ana, noticed your team's sprint velocity increased 18% this month. Here's how to keep climbing..." |
| Onboarding | Encouraging, guided | "Welcome! Let's get your first project set up in under 2 minutes." |

### Writing Principles
1. **Lead with the benefit, not the feature.**
   - Bad: "Our platform supports Kanban boards"
   - Good: "See your entire workflow at a glance with Kanban boards"

2. **Use active voice.**
   - Bad: "Tasks can be assigned to team members"
   - Good: "Assign tasks to team members in one click"

3. **Be specific with numbers.**
   - Bad: "Save a lot of time"
   - Good: "Teams save an average of 6.2 hours per week"

4. **Avoid jargon unless talking to developers.**
   - Bad: "Leverage synergistic paradigms"
   - Good: "Work better together"

5. **Write at an 8th-grade reading level.**
   - Short sentences, active verbs, no unnecessary adjectives.

## Imagery Style

### Photography
- **Subject:** Real diverse teams collaborating (office + remote)
- **Style:** Natural lighting, authentic moments, not staged stock photos
- **Color Treatment:** Slight cool blue tint to match brand palette
- **Aspect Ratios:** 16:9 (web hero), 1:1 (social), 4:5 (Instagram posts)

### Illustrations
- **Style:** Clean geometric shapes, flat design with subtle gradients
- **Colors:** Primary brand palette with neutrals
- **Use:** Feature explanations, empty states, onboarding screens
- **Tools:** Figma with TaskFlow Illustration Kit

### Screenshot Guidelines
- Use dark theme by default
- Remove any real customer data — use placeholder names like "Project Alpha"
- Add subtle drop shadow for depth
- Annotate with brand-colored arrows and highlights

### Icon Style
- Outline icons, 2px stroke
- Rounded corners on all paths
- 24x24 grid
- Consistent visual weight across the set

## Do's and Don'ts

### Brand Voice Do's
- Do use conversational language ("you", "your team")
- Do include specific metrics and data points
- Do reference real scenarios and use cases
- Do keep paragraphs under 4 lines

### Brand Voice Don'ts
- Don't use corporate jargon ("synergy", "leverage", "paradigm")
- Don't overuse exclamation points (max 1 per post)
- Don't claim "the best" without evidence
- Don't use ALL CAPS for emphasis beyond one word

### Visual Do's
- Do maintain consistent margins and spacing
- Do use the primary gradient for hero areas
- Do show the product in context of real work
- Do test all designs in both light and dark mode

### Visual Don'ts
- Don't use warm or orange backgrounds with blue text
- Don't use more than 2 typefaces in any design
- Don't place text on busy imagery without an overlay
- Don't compress or pixelate the logo
